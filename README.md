# CPS-MT
## Code Overview

This repository contains the R scripts used to develop and evaluate a new breeding strategy, **CPS-MT (Cross Potential Selection for Multiple Traits)**, using breeding simulations. The title of this paper is **Cross Potential Selection for Multiple Traits Considering the Progeny Distribution of Future Inbred Lines in Plant Breeding Programs**. The workflow consists of (i) setting up simulation populations and genetic architectures, (ii) running recurrent breeding simulations under CPS-MT and a baseline strategy, and (iii) comparing their performance.

---

### `1.0.function.R`

This script contains a collection of custom functions required throughout the analyses and simulations.  
It includes utility functions for genomic prediction, evaluation of crossing pairs, calculation of progeny genetic (co)variances, selection of crosses and selfing candidates, and generation of figures.

---

### `2.0.simulationSetting.R`

This script defines the simulation settings and generates the initial breeding populations.  
Main tasks include:
- Importing and filtering genotype data (e.g., MAF and LD filtering)
- Constructing marker maps
- Defining genetic architectures by selecting QTNs and assigning marker effects
- Simulating true genetic values and phenotypes under specified heritability and genetic correlation/causality scenarios
- Saving population objects, maps, marker effects, and phenotypic data for downstream analyses

**Note:**  
The directory `midstream/2.0.simulationSetting/` contains preprocessed genome data for the simulations. These files are reused by downstream scripts to ensure consistency across simulation replicates and breeding strategies.

---

### `3.0.CPSMT.R`

This script performs breeding simulations using the proposed strategy **CPS-MT**.  
At each selection cycle, CPS-MT selects crossing pairs by jointly considering:
- the expected mean genetic value of progeny, and  
- the progeny genetic (co)variance (i.e., cross potential) across multiple traits.

---

### `3.1.gridSearch.R`

This script summarizes and compares results from the parameter grid search conducted within `3.0.CPSMT.R`.  
It aggregates simulation outputs across different penalty parameters and is used to identify suitable settings for CPS-MT.

---

### `4.0.PCS.R`

This script performs breeding simulations using the baseline strategy PCS (Penalized Cross Selection).  
PCS selects crossing pairs based on predicted mean performance, with an additional penalty term to discourage crosses that are expected to deviate from the desired range of the secondary trait.  

---

### `4.1.gridSearch.R`

This script summarizes and compares results from the parameter grid search conducted within `4.0.PCS.R`.  

---

### `5.0.comparison.R`

This script compares the simulation outcomes of CPS-MT and PCS.  
It loads results generated by `3.0.CPSMT.R` and `4.0.PCS.R` and produces summary tables and figures, including:
- genetic gain,
- prediction accuracy of genomic estimated breeding values,
- changes in genetic variance and covariance across generations, and

These outputs are used to evaluate the relative performance of the proposed CPS-MT strategy against the baseline PCS approach.
